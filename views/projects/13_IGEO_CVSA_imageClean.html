Road Images Dataset Exploration

For: Corredores Viales S.A.

With: IGEO

Around 2.4 million images were available corresponding to 3,500km of national routes, corresponding to sections I to VI, and the quality of this data set was needed to be known, because it would be used for future applications and analysis.

Process

Merging

The 2.4 million images were grouped into groups of 2000 images, separated by section, route, direction and camera; each group with a metadata csv file.All the images were copied to a single location and the files were merged into one.

Exploration

Correction of incorrect values, due to errors in the data or values ​​in different units. Analysis of grouped information by sections, routes, directions and cameras, to, for example, know the number of images per route. Analysis of information from image files and csv data, and the crossing of these two, for example, to determine missing images.

Report

Charts were generated for the report, along with photos and tables, to facilitate the understanding of the data and results. A report was written detailing the analysis carried out, the results found, and the conclusions reached, proposing actions to be taken.

Result

A report was delivered to the client, with all the findings about the images dataset, and all the actions needed to take in order to clean it and make it useful to the following process of machine learning damage detection

Lessons
Learned: using reference linear sistems to locate images to route. Learned: avoid using to much "fors" for looping. Learned: always end the report with a call to action for each error to fix in the exploration. Errors: the conclution of the report only listed the errors found in the exploration, but give no solution to each one of them. Errors: don't copy files from one computer to another using code (it crashed the server).
