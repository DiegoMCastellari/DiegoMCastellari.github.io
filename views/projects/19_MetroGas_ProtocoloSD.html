SD Protocol

For: Metrogas

With: IGEO

Team:
  • Jorge Volpe - Project Director

Due to a request from ENARGAS (National Gas Regulatory Entity), all the geographic data of facilities, pipelines and stations needed to be delivered in shapefile files with an already designed data structure (known as SD Protocol), with a specific name. The data provided by Metrogas was procesed building an ETL model, which transformed and saved the data from a FGDB to the requiered shapefiles.

Process

Files Preparation

All the data structure requested for each file was created in empty shapefiles. This files were used as data models to create the resulting files. Also a copy of this files were created insede a FGDB, to save the the data resulting from the ETL proceses, before creating the delivered shapefiles.

ETL Process

Jupyter notebooks with all the processing and transformation model were created. Each process read the original Metrogas data, processed it and saved it into the FGDB. The process included renaming columns names and creating new ones, encoding column's values acording to a given domain, cleanning dirty data, overlaying with other layers...

Files Creation

Finally the clean and structured data in the FGDB was saved into shapefiles, created with the structure of the empty ones, and a specific name, containing the name of the company, the name of the object, the date of the delivery, etc.

Lessons
With arcpy, use sanitize_columns=True to keep data structure when saving a Feature Class With arcpy, use the in_memory saving not to create a lot of Feature Classes in the FGDB
